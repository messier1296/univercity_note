# report03 修正意図の整理

このメモは，最初に Codex が作成した `report03.txt`，
現在の `report03.txt`，および現在のソースコードを比較し，
削除・変更された部分について意図を整理したものである。

`report03.txt` 本体は変更していない。

## 全体方針の違い

最初の Codex 版は，課題条件を満たすだけでなく，
不正な入力，部分 write，大きい入力，境界条件などを
広めに扱う方針で書いていた。

現在の版は，授業で扱った内容と課題文の要求に近い範囲へ
絞り，説明とコードを簡潔にする方針になっている。

つまり，Codex 版は「堅牢性と再現性を厚くする」意図が強く，
現在の修正版は「授業課題として自然な実装と説明に寄せる」
意図が強い。

## 現在の版に残っている主な判断

現在のソースコードは，授業課題として以下の判断に寄っている。

- 入力は課題文通りに与えられると仮定する。
- 授業で扱っていない補助関数は増やさない。
- `03.html` のサンプルコードからの変更量を小さくする。
- レポートでは head と tail の共通部分を重複して載せない。
- 実行結果は必要なものを中心にし，詳細な再現コマンドは減らす。

このため，最初の Codex 版よりも安全性や汎用性の説明は減ったが，
課題として読みやすく，元の授業資料との対応が分かりやすい形に
なっている。
