# AGENTS.md

このディレクトリ配下で、ユーザーから `.pdf` を指定されて Markdown 化を求められた場合は、以下の手順で処理する。

## 基本方針

- 出力先は、元PDFと同じディレクトリにある同名の `.md` ファイルとする
- すでに `.md` が存在する場合は、その内容を確認したうえで上書きしてよい
- 目的は単純なテキスト抽出ではなく、講義資料として読める Markdown 構造に直すこと

## 変換手順

1. まず既存の `.md` と `.pdf` の両方があるか確認する
2. PDF本文の確認には `pdftotext -layout <PDF> -` を優先して使う
3. 既存の `.md` がある場合は、それも参照して崩れ方や既存ルールを確認する
4. PDFの内容をもとに、見出し・箇条書き・表を Markdown として再構成する
5. ページ番号やOCRノイズ、図だけのページに由来する不要文字は削る
6. 表として読める箇所は Markdown 表に変換する
7. 手順説明やSPSS出力のような列構造は、読みやすさを優先して表または箇条書きにする

## 穴埋め記号の扱い

- この授業では `「」` が穴埋めを表す
- ユーザーから別指示がない限り、穴埋めの中身は補完しない
- `「」` は Markdown 上では必ず `****` に置き換える
- `「語句」` のように抽出結果に中身が見えていても、ユーザーが自分で埋める前提なら勝手に補完しない
- 文脈上どの位置に穴埋めが入るかだけを整え、順序を復元する

## 構造化ルール

- タイトルは `#`、主要トピックは `##`、その下位項目は `###` を使う
- 箇条書きで足りる内容は無理に文章化しない
- 英語の補足文がある場合は、必要に応じて引用ブロック `>` で分ける
- 数表や検定結果は、可能なら Markdown 表にする
- 図しかないページは、図中の情報が本文で説明されていれば無理に再現しない

## 禁止事項

- 穴埋め語句を推測で埋めない
- OCRの崩れをそのまま残さない
- PDFの見た目を過剰に再現しようとして、読みにくい空白や改行を残さない

## 仕上げ確認

- 見出し階層が不自然でないこと
- `****` の位置が元の穴埋め位置と大きくずれていないこと
- 表が Markdown として崩れていないこと
- 同じディレクトリ内の既存 `.md` があれば、表記ゆれを必要に応じて揃えること
